# AIMall 项目组长总控方案

## 角色定义

本项目采用一个项目组长加四个 agent 的协作方式。

项目组长负责：

- 制定整体产品方案和技术方案。
- 拆分四个 agent 的开发任务。
- 控制接口契约和职责边界。
- 检查每个 agent 的完成度和质量。
- 根据完成情况制定下一轮任务。

四个 agent 只在自己负责的主目录内工作：

| 目录 | 职责 |
| --- | --- |
| `aimall-web` | 用户端 Vue 商城前端 |
| `aimall-admin` | 管理后台前端 |
| `aimall-server` | Java Spring Boot 业务后端 |
| `aimall-ai-service` | Python FastAPI AI 服务 |

## 项目定位

AIMall 是一个带 AI 助手能力的网上购物商城。

商城第一版必须先跑通稳定购物闭环：

1. 用户登录。
2. 商品浏览。
3. 商品详情。
4. 购物车。
5. 创建订单。
6. 模拟支付。
7. 订单查询。

AI 助手不是普通聊天机器人，而是业务型 AI 助手。它必须能结合商城上下文，通过受控工具回答商品、规则、订单和购物车相关问题。

## 架构边界

```text
aimall-web / aimall-admin
        |
        v
aimall-server
        |
        v
aimall-ai-service
        |
        v
大模型 API / RAG / 向量库
```

协作规则：

- `aimall-web` 只能调用 `aimall-server`。
- `aimall-admin` 只能调用 `aimall-server`。
- `aimall-server` 负责调用 `aimall-ai-service`。
- `aimall-ai-service` 如需业务数据，只能调用 `aimall-server` 暴露的内部 AI 接口。
- `aimall-ai-service` 不能直接连接商城数据库。

## 第一版 MVP 范围

商城端：

- 登录。
- 商品列表。
- 商品详情。
- 购物车。
- 订单。
- 模拟支付。

管理端：

- 商品管理。
- 商品分类管理。
- 平台规则和 FAQ 文档管理。
- AI 知识库重建入口。

AI 端：

- 平台规则问答。
- 商品详情问答。
- 商品推荐骨架。
- 订单状态问答骨架。

## 第一版明确不做

- 真实支付。
- 真实物流。
- 秒杀。
- 分销。
- 复杂优惠券引擎。
- 复杂推荐算法。
- 多 Agent AI 流程。
- AI 自动下单。
- AI 自动退款。
- 微服务拆分。
- 分布式事务。

## 全局质量标准

每个 agent 交付时必须说明：

1. 启动命令。
2. 运行端口。
3. 已完成页面或接口。
4. 主要新增或修改文件。
5. 自测结果。
6. 已知未完成事项。
7. 需要项目组长决策的问题。

不能因为“文件存在”就算完成。验收必须看是否能运行、接口是否一致、是否符合任务范围、是否违反禁止事项。
